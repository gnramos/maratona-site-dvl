Na primeira fase e na Final Brasileira da Maratona de Programação usaremos o sistema de submissão eletrônica BOCA.

O sistema de correção das submissões será executado utilizando a distribuição Ubuntu GNU/Linux 20.04 LTS amd64, tendo os seguintes compiladores/interpretadores configurados:

  • C - gcc version 9.3.0 (Ubuntu 9.3.0-17ubuntu1~20.04)
  • C++ - gcc version 9.3.0 (Ubuntu 9.3.0-17ubuntu1~20.04)
  • Python 3 - Python 3.8.5
  • Java - openjdk-11
  • Kotlin - Kotlin version 1.4

O ambiente computacional da Primeira Fase da Maratona de Programação será o mesmo independentemente da sede em que seu time competirá. A programação da primeira fase será centralizada em todo território nacional e ocorrerá de forma virtual e remota. A prova iniciará pontualmente às 14:00 (horário de Brasília) em todas as sedes.

O ambiente computacional é a distribuição Maratona Linux, que é implantada nas sedes pelo boot do pendrive disponibilizado pela organização. Para fins de teste é possível utilizar uma versão pública.

Para testar o Maratona Linux nos computadores da sede é necessário baixar a imagem de pendrive. A imagem deve ser gravada utilizando uma ferramenta como etcher ou dd(1).

É importante notar que o Maratona Linux:

  • DEVE, obrigatoriamente, iniciar direto no metal, ou seja, virtuais não são suportadas;
  • precisa da existência de ao menos uma partição EXT4 ou NTFS com pelo menos 20GB de espaço livre;
  • a opção Secure Boot deve estar desabilitada na BIOS, e;
  • precisa de uma conexão à internet por meio de cabo ethernet e deve ser capaz de receber um IP por DHCP.
